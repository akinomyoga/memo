L2/L3仮想化 (VPN) の色々

IPIP (IP-IP)

RFC 1853 (1995)。 IP-IP (IP in IP Tunneling) は IP 上で IP を実装する。 IP プロトコル番号 94 及び 4 を用いる。

L2F

RFC 2341 (1998)。 L2F (Layer 2 Forwarding) は UDP 1701 上で動作する L2 接続のカプセル化。 Cisco によって提案された。

  • L2F - Wikipedia
  • Layer 2 Forwarding Protocol - Wikipedia
  • 後継: L2TP

PPTP

RFC 2637 (1999)。 PPTP (Point-to-Point Tunneling Protocol) は TCP 上で PPP 通信をカプセル化・暗号化する。 具体的な暗号などの実装は実装者に任されている。 Microsoft によって提案され Windows では標準でサポートされる。 セキュリティ上の問題があり現在では非推奨。

  • 後継: L2TP, SSTP

L2TP/IPsec

[RFC 2661 (1999) (L2TP), RFC 3193 (2001) (L2TP/IPsec)] L2TP (Layer 2 Tunneling Protocol) は UDP 1701 上で L2 接続をカプセル化する。 L2TP 単体では暗号化を提供しない。 L2TP/IPsec は IPsec 上で L2TP 接続を実現する。 因みに IPsec は UDP 500, 4500 上で動作し L2TP はその上に仮想化されるので、 L2TP/IPsec の時には実際の UDP 1701 は使われない。

SSTP (MS-SSTP)

SSTP (Secure Socket Tunneling Protocol) は TLS の上で動作する PPTP の後継。 Microsoft により開発され Windows は標準で対応する。

EtherIP/IPsec

[RFC 3378 (2002)] EtherIP (Tunneling Ethernet Frames in IP Datagrams) も L2TP と似た仕組み。

L2TPv3/IPsec

[RFC 3931 (2005) (L2TPv3)] L2TPv3 は L2TP (L2TPv2) の改良。L2フレーム自体の転送にも対応する。 L2TPv3/IPsec は IPsec 上で L2TPv3 接続を行う。

IKEv1/IPsec

[RFC 2409 (1998) (IKEv1)]

Cisco IPsec

IKEv1/IPsec の改良らしい。

IKEv2/IPsec

[RFC 4306 (2005) (IKEv2)] (巷では VPN のプロトコルであるかの様に記されているが、 調べてみると IPsec の最初の鍵交換のプロトコルの気もする。 結局 IPsec もそうだが何なのか良くわからない。)

OpenVPN

OpenVPN (2001, James Yonan) は VPN ソフトウェア及びそのプロトコルの名前である。 恐らく最も広く使われている SSL-VPN プロトコルである。 既定で TCP/UDP 1194 の上で動く。

SoftEther VPN (PacketiX VPN)

SoftEther は元々、登大遊によって2003年に開発されたソフトウェアである。 現在では PacketiX VPN 及び SoftEther VPN によってサポートされている。 プロトコルはソフトウェアの名称 SoftEther VPN または PacketiX VPN の名で参照される。

AnyConnect (OpenConnect)

AnyConnect は Cisco による SSL-VPN であり、 OpenConnect (2009) はそのオープンソース実装だそうだ。 プロトコル名としては巷では OpenConnect が使われている様だ。

WireGuard

WireGuard (2016, Jason A. Donenfeld) は オープンソースソフトウェアによる SSL-VPN 及び、 そのプロトコルである。

StealthVPN

Astrill によって開発された SSL-VPN のプロトコル。
